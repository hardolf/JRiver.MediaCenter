LyricsFinder

The LyricsFinder looks for lyrics in public lyric web services. Lookup is done for all - or just one at a time - of the current songs in the "Playing Now" list in the JRiver Media Center.
The LyricsFinder can be used as a standalone program and/or as a plug-in for the JRiver Media Center.
The found lyrics are saved in the songs' tags.

Visit the project repository on GitHub: https://github.com/hardolf/JRiver.MediaCenter (LyricsFinder.sln and LyricsFinder subfolder)

Wiki with user manual: https://github.com/hardolf/JRiver.MediaCenter/wiki/LyricsFinder

Bug reports: https://github.com/hardolf/JRiver.MediaCenter/projects/3

Download area with source code and binaries: https://github.com/hardolf/JRiver.MediaCenter/releases

Installation

Before the installation, you must make a couple of decisions:

  • Full or custom installation? i.e. plug-in, standalone and documentation - or just some of them?
  • If more than one JRiver Media Center is installed, which one should have the plug-in? (like in "Highlander": "There can be only one")

Follow these steps:

  1. Go to the release site mentioned above
  2. Get the newest release files to your own local folder
  3. Unpack the setup ZIP file
  4. Look in the ReadMe.txt file
  5. If installing he plug-in, the JRiver Media Center should not be running
  6. Run the Setup.exe file

Track info visualization

The CoverDetailsLyrics track info is a simple track info page for the "Playing Now" view in JRiver Media Center.
It displays - as the name maybe implies - Cover, Details and Lyrics of the song currently being played.

Visit the project repository on GitHub: https://github.com/hardolf/JRiver.MediaCenter (Visualizations/Track info/CoverDetailsLyrics/)

Wiki with user manual: https://github.com/hardolf/JRiver.MediaCenter/wiki/Track-Info-Visualizations

Bug reports: https://github.com/hardolf/JRiver.MediaCenter/projects/3

Download area: https://github.com/hardolf/JRiver.MediaCenter/tree/master/Visualizations/Track%20info/CoverDetailsLyrics

Installation

Follow these steps:

  1. Get the files in the above mentioned GitHub repository folder "CoverDetailsLyrics"
  2. Copy the files to a new folder in your JRiver Media Center program folder, e.g. "C:\Program Files (x86)\J River\Media Center 24\Visualizations\Track Info\CoverDetailsLyrics\" (you decide the last folder name)
  3. Play a tune...That's it!

Version history

The version history outlines the releases since the first version of the LyricsFinder for JRiver Media Center.

The versioning system used is:

  • The first number is used for interface breaking changes.
  • The second number is used for feature changes.
  • The third number is used for bug fixes.

LyricsFinder version 1.2

July 2019. Feature update:

  • Any Media Center playlist can now be selected for finding lyrics, even while other items on the Playing Now list are being played
  • The Media Center Playing Now list is replaced with the current LyricsFinder list when playing from the latter
  • AZLyrics service added
  • Improved playing features from LyricsFinder
  • Configurable automatic version update check added
  • Much faster automatic and manual search with multiple active services, due to concurrency and configurable queuing
  • Separate Plugin and Standalone log files and log4net configuration files
  • Options and lyric services can now be modified from GUI via the menus

Lyric services in this version:

  • Apiseeds Lyrics API - https://apiseeds.com
  • AZLyrics - https://www.azlyrics.com
  • Chart Lyrics - http://www.chartlyrics.com
  • Musixmatch - https://www.musixmatch.com
  • STANDS4 Network Lyrics.com - https://www.lyrics.com

LyricsFinder version 1.1

June 2019. Feature update:

  • Private configuration files are now moved to data directory (%USERPROFILE%\Documents\LyricsFinder\)
  • Configuration of connection to Media Center is now done in GUI
  • Configuration of lyric services is now done in GUI
  • Added context menu and improved menu choices
  • Version update check added
